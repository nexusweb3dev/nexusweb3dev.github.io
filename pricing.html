How It Works

Three steps. No accounts. No API keys. Just HTTP and USDC.

Call the API

Standard REST endpoints. Send your request to any endpoint. The server responds with HTTP 402 Payment Required and a payment specification.

Pay with USDC

Your agent reads the payment details from the 402 response and sends USDC on Base mainnet. Transaction settles in seconds.

Get Results

Retry the original request with the payment proof attached. The server verifies the on-chain payment and returns your data instantly.

Frequently Asked Questions

Why pay-per-call instead of subscriptions?

You pay only for what you use. No monthly minimums, no unused capacity, no billing surprises. Agents can budget per-task and scale usage up or down without contract changes.

What is x402?

x402 builds on HTTP 402 Payment Required — a status code reserved since the early days of HTTP. Your agent calls the API, gets a 402 response with payment details, pays USDC on Base mainnet, and retries with the transaction proof. The entire flow is machine-to-machine with no human intermediary.

Are the protocols open source?

Core protocols are open. The API layer adds convenience, caching, enhanced analysis, and managed infrastructure on top. You can interact with the protocols directly on-chain or use the API for a streamlined experience.

What chains are supported?

Payments are processed on Base mainnet using USDC. Security analysis covers all major EVM chains (Ethereum, Base, Arbitrum, Polygon, BSC, Avalanche) plus Solana. On-chain intelligence endpoints support EVM chains.

How do I integrate?

Two options: direct REST API calls with x402 payment handling, or the MCP wrapper for Claude Code and Cursor integration. SDKs and integration examples are available on GitHub.
